# ADR-001：上下文预算与查询改写策略

| 项 | 内容 |
|----|------|
| 状态 | Accepted |
| 日期 | 2026-07-17 |
| 关联 | PRD §4.2 / §4.3.3；docs/architecture/context-engineering.md, query-rewrite.md |

## 背景

企业客服对话存在：多轮指代、口语同义、历史过长、政策类问题不可胡编。需要在「检索质量」与「成本/延迟/幻觉风险」间取默认策略。

## 决策

1. **上下文分层 L0–L4**，历史默认最多 12 条、单条约 1200 字；staff 镜像为 assistant。  
2. **查询改写默认 L0+L1（规范化+规则同义）**；LLM 改写默认关闭，仅可配置开启并硬超时回落。  
3. **弱证据拒答不调用生成 LLM**；改写不得用于绕过 grounding。  
4. 每次 run 写 **ContextTrace**（日志必选，PG 可选），供监控与 Gap。  
5. Harness 安全阈值与文案 **冷更新**；同义词典可热更新。

## 后果

- 正向：可控、可测、延迟稳定；符合 Honest RAG。  
- 负向：规则词典需运营维护；复杂指代 MVP 能力有限。  
- 后续：会话摘要压缩、HyDE/多查询走 v1.5+ 且独立 eval。

## 备选方案（未采纳）

| 方案 | 未采纳原因 |
|------|------------|
| 每轮全量历史进模型 | 成本高、噪声大、注入面大 |
| 默认 LLM 改写 | 增加 TTFT 与幻觉面；失败模式复杂 |
| 仅向量检索不改写 | 口语/同义召回不足（PRD P3） |
