# 安全与兜底

> 产品对齐：PRD §1.5 原则 1–3、§4.1/4.3.3/4.11、§8.1–8.2。  
> 目标：**宁可降级或拒答，不可胡编、不可裸奔、不可把旁路失败打成主路径成功。**

---

## 1. 分层模型

```
┌─────────────────────────────────────────────────────┐
│ L1  入口安全     鉴权 · 角色 · 限流 · CORS · TLS      │
├─────────────────────────────────────────────────────┤
│ L2  输入安全     空/超长 · 注入 · 非法 role · 脱敏     │
├─────────────────────────────────────────────────────┤
│ L3  决策护栏     路由白名单 · 工具 Registry · Grounding │
├─────────────────────────────────────────────────────┤
│ L4  执行兜底     超时 · 重试策略 · mock/固定话术 · 转人 │
├─────────────────────────────────────────────────────┤
│ L5  输出约束     空输出 · 截断 · 引用自检 · 无密钥泄漏   │
├─────────────────────────────────────────────────────┤
│ L6  数据与审计   密码哈希 · 审计同事务 · 日志 mask      │
└─────────────────────────────────────────────────────┘
```

| 层 | 失败默认 | 用户感知 |
|----|----------|----------|
| L1 | 401/403/429，不进业务 | 登录/稍后重试 |
| L2–L3 | 硬停或强制安全路由 | 固定安全/澄清/拒答文案 |
| L4 | 降级结果 + 标明来源 | 可解释的降级，不装成功 |
| L5 | 兜底话术 / 截断提示 | 仍可读 |
| L6 | 写失败可重试；审计与业务同事务 | 通常无感 |

## 2. 入口安全（L1）

### 2.1 认证与授权

| 规则 | 要求 |
|------|------|
| REST | Bearer JWT（或等价）；过期拒绝 |
| WebSocket | **先连后首帧 auth**；超时未认证关闭；生产 **禁止** token 进 URL query |
| 角色 | `user` / `agent` / `admin`；前端路由 + 后端依赖双检 |
| 生产密钥 | 默认 `SECRET` + 非 development → **拒绝启动** |

### 2.2 限流与暴露面

| 项 | 建议 |
|----|------|
| 用户/IP 限流 | 默认 60/min，可配；429 |
| `/metrics` | 无鉴权 → **仅内网** |
| `/health` | 可对 LB；不回显密钥/内部连接串 |
| 管理 API | 仅 agent/admin；敏感写操作进审计 |

### 2.3 传输与密钥

- TLS 终结在反代（运维）  
- LLM / webhook / 对象存储密钥只进环境变量或密钥管理，禁止写进 Prompt/前端  
- 订单 webhook 失败时的 mock **不得**被当成真实履约数据（UI `data_source`）

## 3. Cognitive Harness（L2–L3 核心）

安全文案与阈值写在**代码常量**，**禁止**进运营 Prompt 模板表。

| 阶段 | 条件 | 动作 | 用户侧 |
|------|------|------|--------|
| prepare | 空问题 | 停止 | 固定提示 |
| prepare | 超长（如 >2000 字） | 停止 | 请缩短 |
| prepare | Prompt 注入 / 控制类请求 | 停止 | 安全拒答（不可运营改） |
| prepare | 历史过长/单条过长 | 裁剪 + flag | 无感或弱提示 |
| prepare | 非法 role | 丢弃；staff→assistant 镜像 | 无感 |
| choose_route | 路由不在白名单 | 强制 `rag` | 正常问答路径 |
| choose_route | 置信度过低 | 强制 `clarify` | 澄清 |
| finalize | 空输出 | 兜底话术 | 可读回复 |
| finalize | 超长输出 | 截断 + 提示 | 部分内容 + 提示 |
| 会话态 | `transferred` | **禁止**调 AI | 只进坐席 |

合法路由集：`{rag, tool, ticket, handoff, clarify}`。

---

## 4. Honest RAG 安全（业务正确性）

| 场景 | 动作 | 是否调生成 LLM |
|------|------|----------------|
| 零命中 | 拒答 + 引导换问/转人 | **否** |
| grounding &lt; 阈值 | 拒答 + 最多 N 条弱来源 | **否** |
| 命中且过阈值 | 流式生成 + 引用 | 是 |
| 引用越界 / 自检失败 | 打标 verification；可降级展示 | 已生成则保留并标 |
| LLM 宕机 | `rag.fallback_llm_down` | 调用失败后 |

**红线：** 不得为提高「解答率」关闭拒答或把无依据问题改写成假相关查询。

## 5. 工具与外部依赖兜底矩阵

| 依赖 | 失败表现 | 策略 | 用户文案要点 |
|------|----------|------|--------------|
| 订单 webhook | 超时 / 5xx | 有限重试 → **mock 或明确失败** + 指标 | 标明「查询通道异常 / 示例数据」 |
| 订单 webhook | 未配置 | 直接 mock | 开发/试点可见 data_source |
| 订单参数 | 无单号 | 槽位追问，不盲调 | 请提供订单号 |
| 检索/向量库 | 不可用 | health 降级；问答走故障话术 | 暂时无法检索 |
| 分类 LLM | 超时/解析失败 | 规则结果或 `faq`+低置信+clarify | 可能多问一句 |
| 改写 LLM | 失败 | 回落原文（规则改写优先） | 无感 |
| 转人工摘要 | 超时 | **空摘要仍入队** | 排队横幅正常 |
| Gap 雷达 / 审计旁路 | 写失败 | best-effort，不挡聊天 | 无感 |
| Handoff 无人认领 | 超时 | 建高优工单 + 会话回 `active` | AI 恢复 + 已建单提示 |

### 5.1 重试红线

| 错误类 | 可重试 | 说明 |
|--------|--------|------|
| 超时 / 5xx | 是（有上限） | 指数退避可选 |
| 4xx 参数 / 鉴权 | **否** | 改参或澄清 |
| 业务去重冲突 | 否 | 返回已有工单 |
| 非 Registry 工具 | **否且不执行** | Harness 拒绝 |

默认：`ORDER_LOOKUP_TIMEOUT_S=5`，工具级重试 ≤2。

## 6. 输出与内容安全（L5）

| 检查 | 行为 |
|------|------|
| 空流式结果 | 固定兜底句 |
| 超 max tokens | 截断 + 「内容过长已截断」 |
| 引用 `[n]` | 与 sources 对齐；越界打标 |
| 模型回显密钥/系统提示 | 尽力过滤；依赖 L2 注入拦截 |
| staff 内部备注 | 永不进用户可见与（错误的）公开 API |

---

## 7. 数据安全与隐私（L6）

| 项 | MVP |
|----|-----|
| 密码 | 强哈希存储 |
| 审计 | actor / action / entity / **脱敏 detail**；与业务同事务 |
| 日志 mask | 默认开：手机、邮箱、订单号部分遮罩 |
| 消息落库脱敏 | 默认关（保 handoff 上下文质量）；可开 |
| 租户 | 单实例隔离，不做伪多租户串数据 |

---

## 8. 兜底优先级（冲突时）

当多条策略同时可触发时，按序：

```
1. 安全硬停（注入 / 未鉴权 / 非法工具）  > 一切业务优化
2. 会话态 transferred                    > 任何 AI 生成
3. Grounding 拒答                        > 流畅瞎答
4. 业务动作成功（建单/入队）              > 摘要/Gap/分析旁路
5. 可解释降级                            > 静默失败
6. 主路径延迟达标                        > 完美可观测写入
```

## 9. MVP 验收清单

- [ ] 错误密码 / 过期 token 不可用  
- [ ] 生产默认 SECRET 拒启  
- [ ] WS 无 token-in-URL  
- [ ] 注入句不进入 RAG 生成  
- [ ] 弱检索不调生成 LLM  
- [ ] webhook 超时有降级且 `data_source` 可辨  
- [ ] handoff 摘要失败仍能排队  
- [ ] handoff 超时建单并回 AI  
- [ ] 日志含手机号样例时已 mask  
- [ ] transferred 用户消息无 token 帧  

---

## 10. 明确不做（避免假安全）

| 不做 | 原因 |
|------|------|
| 开放式无限工具 Agent | 攻击面与副作用不可控 |
| 默认代码沙箱 | 客服场景无刚需 |
| 仅靠 Prompt「请不要泄漏」 | 必须 Harness 硬拦 |
| 语义缓存直接复用政策答案 | 时效与串单风险 |
